Backend server

Motivation

This server shall receive images, and apply some kind of analysis tool using Machine Learning, and give an answer back to the frontend.

Development methodology

This must use Test Driven Development, so any logic code cannot be written unless it is to make a failing test code pass. Why? So we ensure a 100% test coverage.

Key points

Picture representation

Abstractly, we can define a picture as being a tensor of [cols, rows, num_chans]. A black and white picture is thus a [cols, rows] matrix, for which each element of the matrix is a greyscale value. A RGB image shall be [cols, rows, 3], for each color channel. RGB is not the only model of matrix of depth 3 we have, we can have for instance the HSV color space (for Hue, Saturation, Value).

Our backend shall thus expect any kind of picture, and shall convert it into a representation suitable for analysis. Then, the server shall proceed with the analysis, and send back the information to the frontend, which shall display it.
